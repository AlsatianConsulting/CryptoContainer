# Privacy Policy

Effective date: March 2, 2026

CryptoContainer is designed to process encrypted containers and encrypted files locally on your device.

## Data Collection

CryptoContainer does not create accounts, does not require sign-in, and does not collect personal information for analytics, advertising, or profiling.

The app does not operate a remote backend service for normal app use.

## File Handling

Files and containers selected in the app are processed on-device. The app reads and writes files only when you explicitly choose a file, folder, or share action.

Files may be copied temporarily into app-private cache storage when required for:
- opening a shared file
- decrypting or encrypting an AESCrypt file
- exporting a file to another app
- staging keyfiles for VeraCrypt operations

Temporary files are intended to be removed when the operation completes or when the app session ends.

## Credentials and Secrets

Passwords, PIM values, and keyfiles are used only for the requested encryption or decryption operation.

CryptoContainer caches passwords only for the current session. Copied secrets placed on the clipboard by the app are automatically cleared after 30 seconds.

## Sharing

CryptoContainer shares files with other apps only when you explicitly use Android share or open actions. Once you share a file to another app, that app’s handling of the file is governed by its own privacy and security practices.

## Network Use

CryptoContainer does not require a network connection for its core encryption, decryption, or container-management features.

## Third-Party Components

CryptoContainer includes third-party cryptographic and filesystem components needed to support VeraCrypt, NTFS, exFAT, and FAT workflows. Those components are bundled locally inside the app.
